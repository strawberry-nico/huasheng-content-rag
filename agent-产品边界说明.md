# 华胜内容生成 Agent 产品边界说明

## 1. 这个 Agent 是干什么的

这是一个 `To B 内容生产 agent`，不是聊天机器人，也不是备案顾问。

它的核心任务是：

- 把华胜相关业务输入，整理成可用于对外发布的自媒体内容初稿
- 帮市场、销售、业务负责人更快产出口播、图文、标题等内容
- 在表达上做到：专业、可信、可读、不过度宣传

它不负责：

- 替代人工做最终发布判断
- 替代研发、法规、注册、备案专业判断
- 生成内部汇报、投标文件、审评材料、法规咨询意见

## 2. 谁在用

第一使用人群：

- 华胜市场
- 华胜销售
- 懂业务的一线负责人

不建议一开始直接开放给完全不懂业务背景的人自由使用。

## 3. 输出什么

当前输出应聚焦于以下内容类型：

- 口播脚本
- 图文正文
- 标题与封面文案
- 评论区置顶话术
- 朋友圈文案

输出目标不是“文学性”，而是：

- 小改可发
- 风险可控
- 风格统一

## 4. 绝对不能编什么

没有明确知识依据、没有明确可公开权限的信息，默认不能写。

尤其以下内容绝对不能编：

- 客户名称
- 医院名称、医院等级
- 备案号、注册结果、过审结果
- 具体客户案例过程
- 具体产能、收率、节拍、成本、周期等数字收益
- 具体技术参数、验证参数、采样细节、检测值
- 未明确公开的合作关系、项目状态、专家背书
- 资质、认证、法规结论

一句话：

`没有证据，不要补。`

## 5. 风格边界

这是一个 `To B 企业内容 agent`，风格应保持：

- 专业
- 可信
- 克制
- 讲人话
- 懂业务

不应该写成：

- 企业宣传片口号
- 油腻销售文案
- 装专家的术语堆砌
- 夸张承诺式表达
- 假装什么都懂的法规顾问口吻

最接近的目标风格是：

`一个真的懂业务的人，在把问题讲清楚。`

## 6. 遇到模糊输入怎么处理

默认策略不是追问，也不是一次给很多版本，而是：

- 优先保守输出
- 收窄到一个最核心角度
- 不自动补高风险事实

只有在以下场景才值得追问：

- 场景判断冲突明显
- 生成会涉及高风险事实
- 用户明确要求多版本比较

## 7. 前端形态原则

短期内，这个产品不是聊天产品。

当前更适合的是：

- 结构化输入
- 一次生成
- 结果可编辑
- 风险可观察

优先级高于“聊天感”的，是：

- 输出稳定
- 场景不跑偏
- 可人工复核

因此：

- 短期不做开放式对话 agent
- 短期不做长期记忆
- 可以做轻量多轮编辑，但不做自由聊天

## 8. 上线后的人工兜底

上线初期默认采用：

`AI 出初稿，人做最终确认`

人工只需要重点检查 4 件事：

- 场景对不对
- 有没有瞎编
- 有没有不该出现的数字或案例
- 口气像不像华胜

## 9. 当前产品定义

一句话定义这只 Agent：

`华胜市场/销售可用的 To B 自媒体内容初稿生成器。`

它不是万能营销 agent，不是聊天助手，也不是备案顾问。

## 10. 后续迭代原则

后续所有优化，优先级按这个顺序走：

1. 不瞎编
2. 不跑偏
3. 说人话
4. 再谈更强的表现力

如果某个新能力会削弱以上四条，就不应该优先加入。
